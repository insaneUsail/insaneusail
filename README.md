### 🏗️ Featured Engineering Projects

#### 1. 🔍 Review-Codex `TypeScript` `BullMQ` `Redis` `GitHub API`
*An enterprise-grade, asynchronous code analysis agent that handles mass code-review requests via scalable producer-consumer background worker clusters.*
*   **The Problem:** Large-scale repository audits create compute-heavy parsing pipelines that overwhelm traditional HTTP request-response patterns and create performance bottlenecks.
*   **Engineering Highlights:** Built a reliable asynchronously decoupled network inside TypeScript using **BullMQ** and **Redis**. Designed an automated text-chunker that processes incoming streams gracefully under heavy traffic, allowing independent worker pools to handle compute-heavy AI tasks without blocking main API routes. Fully integrated with the GitHub Check Run API to handle payload states natively.
*   **Architecture:** Monolithic Gateway Server $\rightarrow$ Persistent Redis Job Quota Broker $\rightarrow$ Stateless Cluster Compute Workers.

#### 2. 🎙️ AI-interview-WebApp `Next.js` `FastAPI` `Vapi AI` `Zod`
*A real-time, low-latency automated vocal mock interview platform utilizing synchronous voice processing systems.*
*   **The Problem:** Managing conversational state machines and keeping latency low during multi-layered vocal interactions requires strict timing synchronization between client nodes, audio proxies, and dynamic AI backends.
*   **Engineering Highlights:** Orchestrated a clean, modern monorepo separating a responsive Next.js frontend from a robust Python FastAPI ecosystem. Integrated the **Vapi AI voice agent engine** for flawless real-time conversational processing. Enforced strict data validation across both domains using Zod and Pydantic schemas, eliminating parsing runtime failures.
*   **Architecture:** Client Media Matrix $\leftrightarrow$ Next.js Router Proxies $\leftrightarrow$ FastAPI Orchestration Layer $\leftrightarrow$ Vapi AI Voice Network.

#### 3. 📐 Zketch `Next.js` `pnpm-workspaces` `Prisma` `PostgreSQL`
*A type-safe, multi-package collaborative whiteboard ecosystem designed for high-performance visual asset rendering.*
*   **The Problem:** Large mono-applications frequently suffer from bloated builds, weak abstraction boundaries, and untyped state configurations between data layers and layout engines.
*   **Engineering Highlights:** Leveraged **pnpm-workspaces** to configure a modern repository layout with isolated, re-usable internal schema and logic modules. Engineered an intuitive whiteboard rendering engine in Next.js, backed by type-safe database queries via **Prisma ORM** into a scalable **PostgreSQL** cluster.
*   **Architecture:** Distributed Package Monorepo $\rightarrow$ Shared Common Data Schemas $\rightarrow$ Isolated Visual Canvas Layer.
